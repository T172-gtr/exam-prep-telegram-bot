# ExamBot — Telegram-бот для подготовки к экзаменам

Бот для подготовки к экзаменам (ОГЭ, ЕГЭ, МЦКО, диагностика) для учеников 8–11 классов.
Интерфейс: **русский язык**.

---

## Что умеет

- Аккуратное главное меню — пользователь не получает сразу всё подряд.
- Настройка подготовки: класс → экзамен → **несколько предметов** одним набором.
- Несколько активных планов одновременно (по плану на каждый выбранный предмет).
- Настраиваемая рассылка задач: 1/2/3 раза в день в выбранное время.
- Автопроверка ответа: пользователь пишет ответ текстом, бот сравнивает
  с эталоном (`correct_answer` + `acceptable_answers`) и сразу даёт результат
  с объяснением.
- Задания с картинкой (`image_url` / `image_file_id`) и ссылкой на источник.
- Импорт задач из JSON (`/admin_import`) — заготовка под будущий парсер.
- Не повторяет уже выполненные задания, а когда они заканчиваются —
  предлагает сменить предмет/план.

## Быстрый старт

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt

cp .env.example .env
# заполнить BOT_TOKEN и ADMIN_IDS

python main.py
```

При первом запуске:
- создаётся SQLite база `exambot.db`;
- запускаются «мягкие» миграции (добавление колонок к существующим таблицам);
- заполняются справочники, шаблоны планов и примеры заданий;
- запускается планировщик.

## Настройка подготовки (multi-subject)

1. **Класс** — 8, 9, 10, 11.
2. **Тип экзамена** — зависит от класса (ОГЭ/ЕГЭ/МЦКО/Диагностика).
3. **Предметы** — множественный выбор: галочка/▫️ на кнопке. Кнопка
   «✔️ Готово» появляется когда выбран хотя бы один предмет.
4. Для каждого выбранного предмета — **уровень** (низкий/средний/высокий/максимум),
   **минуты в день** (15/30/45/60/90) и **вариант плана** (классический /
   интенсивный / тематический).

Правила:
- Все выбранные предметы относятся к одному и тому же классу и экзамену.
- Один предмет нельзя добавить дважды.
- Сменить класс/экзамен можно только через явный сброс — кнопка
  «🎯 Настроить подготовку» или `/setup` очищает текущие предметы и планы.

## Рассылка

Меню «📅 Рассылка» (или `/schedule`):

1. Выбрать количество рассылок в день — 1, 2 или 3.
2. Для каждой рассылки указать время — выбрать из пресетов
   (07:30, 08:00, 12:00, …) или ввести вручную `HH:MM`.

Планировщик (`scheduler/daily.py`) запускается каждую минуту и отправляет
задание тем пользователям, у которых текущее локальное время (по `TIMEZONE`)
совпадает с любым значением из `users.notify_times`.

В 00:00 (по `TIMEZONE`) сбрасываются дневные счётчики и продвигается
номер текущего дня для всех активных планов.

Дефолт: **1 рассылка в 08:00**.

## Доставка задания и автопроверка

При получении задания бот:

- отправляет картинку, если у задачи есть `image_url`/`image_file_id`;
- запоминает у пользователя `active_task_id` — задачу, на которую ожидается ответ;
- любое следующее **текстовое сообщение** считается ответом и автоматически
  сравнивается с эталоном (`correct_answer`, плюс альтернативы из
  `acceptable_answers`, разделённые `|`);
- сразу пишет «✅ Верно» или «❌ Неверно: …», показывает `explanation`
  (или подсказку, если объяснения нет).

Дополнительно: кнопки `💡 Подсказка`, `👁 Показать ответ`, `🚫 Пропустить`.

Уже выполненные задания пользователю не присылаются повторно. Когда задания
по выбранному предмету и уровню исчерпаны, бот сообщает об этом и предлагает
сменить план/предмет или дождаться пополнения базы.

## Поля модели Task

| Поле | Назначение |
|------|------------|
| `title`, `body` | Заголовок и условие |
| `correct_answer` | Эталонный ответ для автопроверки |
| `acceptable_answers` | Доп. варианты через `\|` |
| `answer` | Legacy-поле, используется как fallback |
| `hint` | Подсказка (`💡`) |
| `explanation` | Подробное объяснение (показывается после ответа) |
| `image_url` | URL картинки (бот отправит фото) |
| `image_file_id` | Telegram file_id картинки (приоритет над URL) |
| `source_url` | Ссылка на источник задания |
| `target_level` | `low` / `medium` / `high` / `max` |
| `tags` | comma-separated теги |

## Импорт заданий

### Через `/admin_add_task`

Пошаговый FSM в боте: предмет → уровень → заголовок → условие → ответ
(можно перечислить варианты через `|`) → подсказка → подтверждение.

### Через JSON-файл `/admin_import`

Положи файл в `data/tasks_import.json` (или укажи путь через переменную
окружения `TASKS_IMPORT_PATH`), пример — `data/tasks_import.example.json`:

```json
[
  {
    "exam_code": "oge",
    "subject_code": "math",
    "target_level": "low",
    "title": "Проценты",
    "body": "Найдите 20% от 150.",
    "correct_answer": "30",
    "acceptable_answers": "30.0",
    "hint": "20% = 0.2",
    "explanation": "150 · 0.2 = 30",
    "image_url": "https://example.com/img.png",
    "source_url": "https://example.com/src/1"
  }
]
```

Дубликаты по `(subject_id, title)` пропускаются. Реальный парсер (например,
скачивание с сайтов ФИПИ) подключается через одну функцию
`import_tasks(session, items)` в `db/import_tasks.py`.

## Подписка

Заглушка без реального платёжного шлюза. Под кнопкой «оплатить» создаётся
запись `Payment(status="success", provider_tx_id="STUB-N")` и активируется
`Subscription`. В `/admin` показывается сводка: активных Premium, разбивка
по тарифам, число и сумма успешных платежей.

## База данных

SQLite (по умолчанию), таблицы создаются автоматически. Для добавления
новых колонок к существующим таблицам в `db/database.py` реализованы
лёгкие миграции (`ALTER TABLE … ADD COLUMN`). Для production используйте
Alembic.

| Таблица | Что хранит |
|---------|------------|
| `grade_levels`, `exam_types`, `subjects` | Справочники |
| `plan_templates` | Шаблоны планов |
| `tasks` | Банк заданий (с полями для медиа и автопроверки) |
| `users` | Пользователи + настройки рассылки (`notify_count`, `notify_times`) + `active_task_id` |
| `user_subjects` | Выбранные предметы (множественный выбор) |
| `user_plans` | Активные планы (по плану на предмет) |
| `user_progress` | Ответы пользователя |
| `subscriptions`, `payments` | Подписки и платежи (заглушка) |

## Ограничения MVP

- Платёжный шлюз — заглушка (`status="success"`).
- Веб-парсер задач не реализован; есть только импорт из JSON.
- Миграции БД минимальные (только добавление колонок) — для prod нужен Alembic.
- Все рассылки выполняются в одном часовом поясе из `TIMEZONE` (нет
  индивидуальных таймзон у пользователей).
- Бот работает в режиме long polling; для webhook нужно заменить
  `start_polling` в `main.py`.
